Foire Aux Questions

Trouvez les réponses aux questions les plus fréquentes sur Coclico. Utilisez les filtres de catégorie pour naviguer rapidement.

Installation

Quelle est la configuration requise pour Coclico ?
  • OS : Windows 10 21H2 (64 bits) ou Windows 11
  • RAM : 4 Go minimum, 8 Go recommandé (pour le module IA)
  • Disque : 500 Mo pour l'application + ~4 Go pour le modèle GGUF si vous utilisez l'IA
  • Permissions : Compte administrateur — Coclico demande une élévation UAC au démarrage
  • GPU (optionnel) : Carte compatible Vulkan pour accélérer l'inférence IA locale

Le runtime .NET 10 est inclus dans l'installeur — rien d'autre à installer.

Windows bloque l'installeur avec SmartScreen. Est-ce normal ?

Oui, c'est normal pour un logiciel dont la signature de code n'est pas encore reconnue par Microsoft. Coclico est open-source et vous pouvez inspecter son code source sur GitHub. Pour continuer :

  1. Cliquez sur « Informations complémentaires » dans la fenêtre SmartScreen
  2. Cliquez sur « Exécuter quand même »

Alternativement, compilez Coclico directement depuis les sources avec dotnet build après avoir cloné le dépôt.

Puis-je installer Coclico sans droits administrateur ?

Non. Coclico nécessite des droits administrateur pour accéder aux API Windows bas niveau : WMI, mémoire noyau (ntdll), registre système et gestion des processus. L'élévation UAC est demandée une seule fois au démarrage. Une version « mode utilisateur » à fonctionnalités réduites est prévue pour une version future.

Utilisation

Le RAM Cleaner peut-il endommager mon système ?

Non. Le RAM Cleaner utilise les API Windows officielles pour réduire les working sets des processus et vider les listes standby — la même opération que Windows effectue en interne sous pression mémoire. Les applications en cours continuent normalement. Les pages mémoire réduites sont paginées sur disque et rechargées à la demande. L'effet visible est une réduction immédiate de la RAM utilisée avec une légère augmentation temporaire des I/O disque si des applications sont actuellement inactives.

Le Nettoyage profond peut-il supprimer des fichiers importants ?

Coclico supprime uniquement des catégories de fichiers connues comme sûres (temp, cache navigateur, logs, corbeille, prefetch, etc.). Aucun document personnel ni fichier système critique n'est ciblé. Chaque catégorie peut être activée/désactivée individuellement avant l'exécution. Un rapport détaillé liste les fichiers à supprimer avant toute action.

Comment configurer la couleur d'accentuation de l'interface ?

Allez dans Paramètres → Apparence. La couleur d'accentuation par défaut est #6366F1 (Indigo). Saisissez n'importe quelle valeur hexadécimale ou choisissez parmi les préréglages. Les modifications sont appliquées immédiatement sans redémarrage via ThemeService qui met à jour dynamiquement les ressources WPF.

IA Locale

Mes données sont-elles envoyées dans le cloud lors de l'utilisation de l'IA ?

Non, absolument pas. Coclico applique un principe Zero Cloud : toute l'inférence IA tourne localement sur votre machine via LLamaSharp 0.26 avec un modèle GGUF. Aucune requête réseau n'est effectuée pour l'IA. Vos données système, conversations et Flow Chains ne quittent jamais votre machine.

Quel modèle GGUF est recommandé pour Coclico IA ?

Coclico fonctionne avec des modèles GGUF quantifiés (Q4_K_M ou Q5_K_M). Modèles recommandés :

  • Llama 3.2 3B Q5_K_M — léger, rapide sur CPU, bonne qualité pour les tâches système
  • Mistral 7B Q4_K_M — meilleure qualité, nécessite 8 Go RAM
  • Phi-3 Mini 3.8B Q5 — excellent rapport performance/qualité sur CPU

Placez le fichier dans %APPDATA%\Coclico\resource\model\ et nommez-le IA-support-chat.gguf. L'accélération GPU Vulkan est automatique si supportée.

Qu'est-ce que le mode « AutoPatcher Audit uniquement » ?

L'AutoPatcher est le système autonome de Coclico qui peut proposer des patches de code pour s'améliorer lui-même (Phase 3.4). En mode Audit uniquement (par défaut), les patches générés par l'IA sont consignés dans le journal d'audit mais jamais appliqués automatiquement. Vous devez les approuver manuellement via ApproveAndApplyAsync(). Chaque patch est validé par le Digital Twin (vérification de la complexité cyclomatique) et un snapshot de rollback est créé avant toute écriture disque.

Flow Chains

Combien de nœuds peut contenir une Flow Chain ?

Il n'y a pas de limite stricte dans le code. En pratique, les chaînes de plus de 100 nœuds deviennent difficiles à maintenir. Coclico fournit 28 types de nœuds (NodeType enum). Pour les automatisations complexes, créez plusieurs chaînes modulaires et reliez-les via des appels inter-chaînes.

Que se passe-t-il si un nœud échoue dans une Flow Chain ?

Le comportement dépend de la propriété OnErrorAction configurée sur chaque nœud :

  • ContinueNext (par défaut) — l'erreur est consignée, l'exécution continue avec le nœud suivant
  • StopChain — l'exécution s'arrête immédiatement, aucun nœud suivant ne s'exécute
  • SkipToEnd — tous les nœuds intermédiaires sont sautés, seul le dernier nœud de la chaîne s'exécute
Les Flow Chains peuvent-elles accéder au registre Windows ?

Oui, via les nœuds SetRegistryValue et DeleteRegistryKey. Cependant, chaque opération passe par WorkflowPipelineExecutionService qui applique ISecurityPolicy avant l'exécution. Par défaut, les modifications de HKLM\SYSTEM\CurrentControlSet et d'autres clés sensibles sont bloquées. Personnalisez la politique via %APPDATA%\Coclico\security-policy.json.

Performance

Coclico ralentit-il mon PC en utilisation normale ?

Non. En mode inactif (icône dans la barre système), Coclico consomme moins de 20 Mo RAM et moins de 0,1% CPU. Le service de télémétrie (DynamicTracerService) échantillonne le système une fois par seconde avec un overhead négligeable. L'OptimizationEngine tourne toutes les 30 secondes en utilisant des buffers ArrayPool pour éviter les allocations GC.

Le module IA gèle-t-il l'interface pendant l'inférence ?

Non. Coclico utilise deux contextes LLM entièrement isolés (dual-executor) : un pour le chat utilisateur, un pour l'OptimizationEngine en arrière-plan. Le thread UI ne bloque jamais en attendant une réponse LLM — les résultats sont retournés via async/await et les mises à jour UI utilisent Dispatcher.InvokeAsync. L'inférence peut prendre 5 à 30 secondes selon le modèle et le matériel, sans jamais geler l'interface.

Comment réduire la consommation de ressources de Coclico ?

Plusieurs options dans Paramètres → Général :

  • Désactiver le rafraîchissement automatique du Dashboard (passer de 1s à 5s)
  • Réduire la fréquence de l'OptimizationEngine (60s au lieu de 30s)
  • Désactiver le module IA si inutilisé (décharge le modèle GGUF de la mémoire)
  • Désactiver les notifications toast pour réduire les redessinages UI
