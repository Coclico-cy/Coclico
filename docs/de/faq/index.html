Finden Sie Antworten auf die häufigsten Fragen zu Coclico. Verwenden Sie die Kategorie-Filter für eine schnelle Navigation.

Installation

Was sind die Systemanforderungen für Coclico?
  • Betriebssystem: Windows 10 21H2 (64-Bit) oder Windows 11
  • RAM: Mindestens 4 GB, 8 GB empfohlen (für das KI-Modul)
  • Festplatte: 500 MB für die App + ca. 4 GB für das GGUF-Modell bei KI-Nutzung
  • Berechtigungen: Administratorkonto — Coclico fordert beim Start eine UAC-Erhöhung an
  • GPU (optional): Vulkan-kompatible Karte zur Beschleunigung der lokalen KI-Inferenz

Die .NET 10-Laufzeitumgebung ist im Installer enthalten — es muss nichts weiteres installiert werden.

Windows blockiert den Installer mit SmartScreen. Ist das normal?

Ja, das ist normal für Software, deren Code-Signatur von Microsoft noch nicht erkannt wird. Coclico ist Open-Source und Sie können den Quellcode auf GitHub einsehen. Zum Fortfahren:

  1. Klicken Sie im SmartScreen-Fenster auf "Weitere Informationen"
  2. Klicken Sie auf "Trotzdem ausführen"
Kann ich Coclico ohne Administratorrechte installieren?

Nein. Coclico benötigt Administratorrechte für den Zugriff auf Windows-APIs auf niedriger Ebene: WMI, Kernel-Speicher (ntdll), Systemregistrierung und Prozessverwaltung. Die UAC-Erhöhung wird einmalig beim Start angefordert.

Nutzung

Kann der RAM Cleaner mein System beschädigen?

Nein. Der RAM Cleaner verwendet offizielle Windows-APIs, um Working Sets von Prozessen zu verkleinern und Standby-Listen des Kernels zu leeren — dieselbe Operation, die Windows intern bei Speicherdruck durchführt. Laufende Anwendungen arbeiten weiterhin normal. Freigabeergebnisse liegen typischerweise zwischen 200 und 800 MB.

Kann Deep Clean wichtige Dateien löschen?

Coclico löscht nur bekannte sichere Dateikategorien (Temp, Browser-Cache, Protokolle, Papierkorb, Prefetch usw.). Keine persönlichen Dokumente oder kritischen Systemdateien werden angesteuert. Jede Kategorie kann einzeln aktiviert/deaktiviert werden. Ein detaillierter Bericht listet Dateien vor dem Löschen auf.

Wie ändere ich die Akzentfarbe der Oberfläche?

Gehen Sie zu Einstellungen → Erscheinungsbild. Die Standard-Akzentfarbe ist #6366F1 (Indigo). Geben Sie einen beliebigen Hex-Wert ein oder wählen Sie aus Voreinstellungen. Änderungen werden sofort ohne Neustart über ThemeService angewendet.

Lokale KI

Werden meine Daten in die Cloud gesendet, wenn ich die KI verwende?

Nein, absolut nicht. Coclico setzt das Zero Cloud-Prinzip durch: Alle KI-Inferenz läuft lokal auf Ihrem Computer über LLamaSharp 0.26 mit einem GGUF-Modell. Für KI werden keine Netzwerkanfragen gestellt. Ihre Systemdaten, Gespräche und Flow Chains verlassen nie Ihren Computer.

Welches GGUF-Modell wird für Coclico KI empfohlen?

Coclico funktioniert mit quantisierten GGUF-Modellen (Q4_K_M oder Q5_K_M). Empfohlene Modelle:

  • Llama 3.2 3B Q5_K_M — leicht, schnell auf CPU
  • Mistral 7B Q4_K_M — bessere Qualität, benötigt 8 GB RAM
  • Phi-3 Mini 3.8B Q5 — ausgezeichnetes Leistungs-/Qualitätsverhältnis auf CPU

Legen Sie die Datei in %APPDATA%\Coclico\resource\model\ ab und benennen Sie sie IA-support-chat.gguf.

Was ist der "AutoPatcher Nur-Audit"-Modus?

Der AutoPatcher ist Cocliicos autonomes System, das Code-Patches zur Selbstverbesserung vorschlagen kann (Phase 3.4). Im Nur-Audit-Modus (Standard) werden KI-generierte Patches im Audit-Protokoll erfasst, aber nie automatisch angewendet. Sie müssen sie manuell über ApproveAndApplyAsync() genehmigen. Jeder Patch wird vom Digital Twin validiert und ein Rollback-Snapshot wird vor jedem Schreiben auf Festplatte erstellt.

Flow Chains

Wie viele Knoten kann eine Flow Chain enthalten?

Es gibt keine strenge Grenze im Code. In der Praxis werden Ketten mit mehr als 100 Knoten schwer zu warten. Coclico bietet 28 Knotentypen (NodeType). Für komplexe Automatisierungen erstellen Sie mehrere modulare Ketten.

Was passiert, wenn ein Knoten in einer Flow Chain fehlschlägt?

Das Verhalten hängt von der OnErrorAction-Eigenschaft jedes Knotens ab:

  • ContinueNext (Standard) — Fehler protokollieren, mit dem nächsten Knoten fortfahren
  • StopChain — Ausführung sofort beenden
  • SkipToEnd — alle Zwischenknoten überspringen, nur den letzten Knoten ausführen
Können Flow Chains auf die Windows-Registrierung zugreifen?

Ja, über die Knoten SetRegistryValue und DeleteRegistryKey. Jede Operation wird jedoch durch den WorkflowPipelineExecutionService geleitet, der die ISecurityPolicy anwendet. Standardmäßig sind Änderungen an HKLM\SYSTEM\CurrentControlSet und anderen sensiblen Schlüsseln blockiert.

Leistung

Verlangsamt Coclico meinen PC im normalen Betrieb?

Nein. Im Standby-Modus (Taskleistensymbol) verbraucht Coclico weniger als 20 MB RAM und unter 0,1 % CPU. Der DynamicTracerService sampelt das System einmal pro Sekunde mit vernachlässigbarem Overhead. Die OptimizationEngine läuft alle 30 Sekunden mit ArrayPool-Puffern zur Vermeidung von GC-Allokationen.

Friert das KI-Modul die Oberfläche während der Inferenz ein?

Nein. Coclico verwendet zwei vollständig isolierte LLM-Kontexte (Dual-Executor): einen für den Benutzer-Chat, einen für den Hintergrund-OptimizationEngine. Der UI-Thread blockiert nie auf eine LLM-Antwort — Ergebnisse werden über async/await zurückgegeben und UI-Updates verwenden Dispatcher.InvokeAsync.

Wie kann ich den Ressourcenverbrauch von Coclico reduzieren?

Optionen in Einstellungen → Allgemein:

  • Automatische Dashboard-Aktualisierung deaktivieren (von 1s auf 5s)
  • OptimizationEngine-Frequenz reduzieren (60s statt 30s)
  • KI-Modul deaktivieren wenn nicht benötigt (entlädt GGUF-Modell aus Speicher)
  • Toast-Benachrichtigungen deaktivieren um UI-Neuzeichnungen zu reduzieren
