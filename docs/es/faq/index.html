Preguntas Frecuentes

Encuentra respuestas a las preguntas más comunes sobre Coclico. Usa los filtros de categoría para navegar rápidamente.

Instalación Uso IA Local Flow Chains Rendimiento

Instalación

¿Cuáles son los requisitos del sistema para Coclico?
  • SO: Windows 10 21H2 (64 bits) o Windows 11
  • RAM: 4 GB mínimo, 8 GB recomendado (para el módulo IA)
  • Disco: 500 MB para la app + ~4 GB para el modelo GGUF si usas IA
  • Permisos: Cuenta de administrador — Coclico solicita elevación UAC al arrancar
  • GPU (opcional): Tarjeta compatible Vulkan para acelerar la inferencia de IA local

El runtime .NET 10 está incluido en el instalador — no hay que instalar nada más.

Windows bloquea el instalador con SmartScreen. ¿Es normal?

Sí, es normal para software cuya firma de código aún no está reconocida por Microsoft. Coclico es de código abierto y puedes inspeccionar su código fuente en GitHub. Para continuar:

  1. Haz clic en "Más información" en la ventana de SmartScreen
  2. Haz clic en "Ejecutar de todas formas"

Alternativamente, compila Coclico directamente desde el código fuente usando dotnet build tras clonar el repositorio.

¿Puedo instalar Coclico sin derechos de administrador?

No. Coclico requiere derechos de administrador para acceder a las APIs de Windows de bajo nivel: WMI, memoria del kernel (ntdll), registro del sistema y gestión de procesos. La elevación UAC se solicita una sola vez al arrancar. Una versión con funcionalidad reducida en "modo usuario" está prevista para una versión futura.

Uso

¿Puede el RAM Cleaner dañar mi sistema?

No. El RAM Cleaner usa las APIs oficiales de Windows para reducir los working sets de los procesos y vaciar las listas standby — la misma operación que Windows realiza internamente bajo presión de memoria. Las aplicaciones en ejecución continúan con normalidad. Las páginas de memoria recortadas se paginan al disco y se recargan a demanda. El efecto visible es una reducción inmediata de la RAM usada con un ligero aumento temporal de E/S de disco si las aplicaciones están inactivas.

¿Puede la Limpieza Profunda borrar archivos importantes?

Coclico solo elimina categorías de archivos conocidas como seguras (temp, caché de navegador, logs, papelera, prefetch, etc.). No se apuntan documentos personales ni archivos críticos del sistema. Cada categoría puede activarse/desactivarse individualmente antes de ejecutar. Un informe detallado lista los archivos a eliminar antes de realizar ninguna acción.

¿Cómo configuro el color de acento de la interfaz?

Ve a Configuración → Apariencia. El color de acento predeterminado es #6366F1 (Índigo). Introduce cualquier valor hexadecimal o elige entre los presets. Los cambios se aplican inmediatamente sin reinicio mediante ThemeService que actualiza dinámicamente los recursos WPF.

IA Local

¿Se envían mis datos a la nube al usar la IA?

No, en absoluto. Coclico aplica el principio Zero Cloud: toda la inferencia de IA se ejecuta localmente en tu máquina via LLamaSharp 0.26 con un modelo GGUF. No se realiza ninguna petición de red para la IA. Tus datos del sistema, conversaciones y Flow Chains nunca salen de tu máquina.

¿Qué modelo GGUF se recomienda para Coclico IA?

Coclico funciona con modelos GGUF cuantizados (Q4_K_M o Q5_K_M). Modelos recomendados:

  • Llama 3.2 3B Q5_K_M — ligero, rápido en CPU, buena calidad para tareas del sistema
  • Mistral 7B Q4_K_M — mejor calidad, requiere 8 GB RAM
  • Phi-3 Mini 3.8B Q5 — excelente relación rendimiento/calidad en CPU

Coloca el archivo en %APPDATA%\Coclico\resource\model\ y nómbralo IA-support-chat.gguf. La aceleración GPU Vulkan es automática si es compatible.

¿Qué es el modo "AutoPatcher Solo Auditoría"?

El AutoPatcher es el sistema autónomo de Coclico que puede proponer parches de código para mejorarse a sí mismo (Fase 3.4). En modo Solo Auditoría (por defecto), los parches generados por IA se registran en el log de auditoría pero nunca se aplican automáticamente. Debes aprobarlos manualmente via ApproveAndApplyAsync(). Cada parche es validado por el Digital Twin (verificación de complejidad ciclomática) y se crea un snapshot de rollback antes de cualquier escritura en disco.

Flow Chains

¿Cuántos nodos puede tener una Flow Chain?

No hay un límite estricto en el código. En la práctica, las cadenas de más de 100 nodos se vuelven difíciles de mantener. Coclico proporciona 28 tipos de nodos (enum NodeType). Para automatizaciones complejas, crea múltiples cadenas modulares y enlázalas mediante llamadas entre cadenas.

¿Qué ocurre si un nodo falla en una Flow Chain?

El comportamiento depende de la propiedad OnErrorAction configurada en cada nodo:

  • ContinueNext (por defecto) — el error se registra, la ejecución continúa con el siguiente nodo
  • StopChain — la ejecución se detiene inmediatamente, ningún nodo posterior se ejecuta
  • SkipToEnd — todos los nodos intermedios se omiten, solo se ejecuta el último nodo de la cadena
¿Pueden las Flow Chains acceder al registro de Windows?

Sí, mediante los nodos SetRegistryValue y DeleteRegistryKey. Sin embargo, cada operación pasa por WorkflowPipelineExecutionService que aplica ISecurityPolicy antes de la ejecución. Por defecto, las modificaciones de HKLM\SYSTEM\CurrentControlSet y otras claves sensibles están bloqueadas. Personaliza la política via %APPDATA%\Coclico\security-policy.json.

Rendimiento

¿Ralentiza Coclico mi PC durante el uso normal?

No. En modo inactivo (icono en la bandeja del sistema), Coclico consume menos de 20 MB de RAM y menos del 0,1% de CPU. El servicio de telemetría (DynamicTracerService) muestrea el sistema una vez por segundo con un overhead negligible. El OptimizationEngine se ejecuta cada 30 segundos usando buffers ArrayPool para evitar asignaciones GC.

¿El módulo IA congela la interfaz durante la inferencia?

No. Coclico usa dos contextos LLM completamente aislados (dual-executor): uno para el chat de usuario, otro para el OptimizationEngine en segundo plano. El hilo de UI nunca bloquea esperando una respuesta LLM — los resultados se devuelven via async/await y las actualizaciones de UI usan Dispatcher.InvokeAsync. La inferencia puede tardar entre 5 y 30 segundos según el modelo y el hardware, sin congelar nunca la interfaz.

¿Cómo puedo reducir el consumo de recursos de Coclico?

Varias opciones en Configuración → General:

  • Desactivar la actualización automática del Panel de Control (cambiar de 1s a 5s)
  • Reducir la frecuencia del OptimizationEngine (60s en lugar de 30s)
  • Desactivar el módulo IA si no se usa (descarga el modelo GGUF de la memoria)
  • Desactivar las notificaciones toast para reducir los redibujos de la UI
