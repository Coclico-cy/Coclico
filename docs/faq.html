Installation & Démarrage 5 questions

Plusieurs fonctionnalités de Coclico nécessitent des privilèges élevés :

  • RAM Cleaner — Les opérations NtSetSystemInformation (purge Standby, Working Sets) requièrent SeProfileSingleProcessPrivilege
  • Processus système — Surveiller et terminer des processus système protégés
  • Registre HKLM — Lire les clés d'installation de toutes les apps (incluant les apps installées machine)
  • Démarrage Windows — Modifier les clés de registre Run pour le démarrage auto

L'UAC est demandé dès le lancement via app.manifest avec requireAdministrator. Si vous ne cliquez pas Oui, Coclico se relance automatiquement avec la demande d'élévation.

Oui, mais avec une version minimale requise. Coclico cible Windows 10 22H2 (build 22621) ou supérieur, ainsi que Windows 11 toutes versions.

La raison de cette limitation est l'utilisation d'APIs Windows introduites dans cette version, notamment pour les opérations de surveillance réseau et certains APIs WPF-UI 4.2.

Vérifiez votre version : Win+R → winver → le numéro de build doit être ≥ 22621.

Non. Coclico est distribué en mode self-contained (--self-contained). Le runtime .NET 10 est inclus dans l'archive ZIP. Aucune installation supplémentaire n'est requise.

La contrepartie est une taille d'archive plus grande (~250-300 MB), mais l'application fonctionnera sur tout Windows 10 22H2+ sans pré-requis.

Toutes les données sont dans : %AppData%\Coclico\

  • settings.json — Paramètres
  • flow_chains.json — Vos automatisations
  • custom_apps_data.json — Noms et catégories personnalisés
  • manual_apps.json — Apps ajoutées manuellement
  • logs\ — Journaux (rotation 14 jours)

Pour sauvegarder votre configuration, copiez simplement ce dossier. Pour migrer vers un autre PC, collez-le au même chemin.

Coclico est portable — pas d'entrée dans Ajout/Suppression de programmes. Pour désinstaller :

  1. Fermez Coclico complètement (clic droit sur l'icône tray → Quitter)
  2. Désactivez le démarrage automatique dans les Paramètres Coclico (ou via Démarrage du Gestionnaire des tâches)
  3. Supprimez le dossier d'installation de Coclico
  4. Optionnel : supprimez %AppData%\Coclico\ pour effacer toutes les données

Flow Chains 5 questions

Oui, c'est précisément l'intérêt. Les raccourcis globaux utilisent un hook clavier bas niveau (WH_KEYBOARD_LL) qui écoute les entrées clavier à l'échelle du système, quel que soit l'application en premier plan.

Coclico peut être minimisé dans le tray, voire invisible — les raccourcis fonctionnent toujours tant que le processus est actif.

Chaque nœud dispose de paramètres Retry et Timeout indépendants :

  • Retry : nombre de tentatives supplémentaires en cas d'échec (0 = pas de retry)
  • Timeout : durée maximale avant abandon du nœud (0 = pas de limite)

Si un nœud échoue après tous ses retries, l'exécution s'arrête à ce point et un log d'erreur est généré. Les nœuds suivants ne sont pas exécutés.

Dans %AppData%\Coclico\flow_chains.json — un fichier JSON lisible. Vous pouvez l'ouvrir avec n'importe quel éditeur de texte. La sauvegarde est automatique à chaque modification.

Pour l'instant, il faut partager manuellement le fichier flow_chains.json ou copier l'entrée JSON correspondant à la chaîne. Une fonctionnalité d'import/export dédiée est prévue dans une version future.

Coclico AI 5 questions

Absolument non. L'IA de Coclico tourne entièrement en local sur votre machine via LLamaSharp. Le modèle GGUF est chargé en mémoire et l'inférence se fait sur votre CPU (ou GPU si configuré). Aucune requête réseau n'est effectuée pour l'IA.

Pour une utilisation optimale avec Coclico :

  • Recommandé : Llama 3.2 3B Instruct Q4_K_M (~2 GB, rapide)
  • Qualité supérieure : Mistral 7B Instruct Q4_K_M (~4.5 GB)
  • Machines limitées : Phi-3.5 Mini Instruct Q4 (~2 GB)

Le fichier doit être renommé exactement IA-support-chat.gguf et placé dans resource/model/.

Plusieurs solutions :

  • Modèle plus petit : utilisez un modèle 3B plutôt que 7B
  • Quantisation plus agressive : choisissez Q2_K ou Q3_K plutôt que Q4_K
  • Accélération GPU : compilez avec le backend LLamaSharp.Backend.Vulkan (nécessite une recompilation)
  • Plus de RAM : avec 16 GB+, le modèle est chargé entièrement en RAM, évitant les swaps

L'IA peut naviguer entre les modules de Coclico (ouvrir le Dashboard, le RAM Cleaner, etc.) et déclencher des actions comme un nettoyage RAM. Elle ne peut pas encore ouvrir des applications tierces — cette fonctionnalité est possible via les Flow Chains.

Non. Le module AI est présent dans l'interface mais affiche un message indiquant que le modèle est absent si resource/model/IA-support-chat.gguf n'existe pas. Tous les autres modules de Coclico fonctionnent parfaitement sans IA.

Performance & RAM 4 questions

Oui, temporairement avec le Profil Profond. Vider la liste Standby et désactiver SuperFetch force Windows à recharger les données depuis le disque lors des premiers accès suivant le nettoyage. Cette "période de réchauffement" dure généralement 30-60 secondes.

Pour un usage quotidien, le Profil Rapide (Working Sets + GC .NET) est plus approprié — il est rapide et sans impact notable sur les performances.

  • RAM physique — Votre mémoire installée (barettes DDR). Rapide, limitée physiquement.
  • Mémoire virtuelle (Commit) — Espace d'adressage total alloué par tous les processus. Peut dépasser la RAM physique car Windows optimise les pages non utilisées.
  • Pagefile — Fichier d'échange sur le disque dur/SSD. Utilisé quand la RAM physique est saturée. Beaucoup plus lent que la RAM.

Le RAM Cleaner surveille les trois pour vous donner une vue complète de la santé mémoire de votre système.

Recommandations :

  • Par seuil : 80-85% est généralement le bon équilibre. En dessous, le nettoyage est inutile. Au-dessus de 90%, des ralentissements sont déjà perceptibles.
  • Par intervalle : toutes les 30-60 minutes en usage intensif, ou désactivé en usage bureautique léger.

Le Profil Rapide est recommandé pour l'auto-clean pour minimiser l'impact système.

C'est normal. Windows gère activement la mémoire — il précharge des données (SuperFetch/SysMain), libère et réalloue en permanence. Les fluctuations reflètent l'activité normale du gestionnaire de mémoire Windows, pas un problème.

Une tendance à la hausse continue sur 90 secondes sans retour indique une possible fuite mémoire dans une application.

Développeurs & Contribution 4 questions

Le code source est disponible sur GitHub. Consultez le Wiki Développeur pour comprendre l'architecture et commencer à contribuer.

Le processus complet est documenté dans le Wiki Développeur → Ajouter un nœud. En résumé :

  1. Déclarer le type dans FlowLink.cs
  2. Implémenter l'exécution dans FlowChainExecutionService.cs
  3. Ajouter l'UI dans FlowChainsView.xaml
  4. Écrire les tests dans Coclico.Tests

Le projet Coclico.Tests utilise xUnit + Moq. Lancez les tests avec :

dotnet test Coclico.Tests/Coclico.Tests.csproj

Ajoutez des fichiers .md dans resource/docs/. Le RagService indexe automatiquement tous les fichiers Markdown de ce dossier au démarrage. Pas de recompilation nécessaire — il suffit de relancer Coclico.
